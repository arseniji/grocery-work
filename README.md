ТЗ проекта


Выполняется в командах 3-4 человека. Выбрать предметную область и подготовить схему данных в 3 нормальной форме на 3-5 таблиц. Не больше и не меньше. Разработать веб приложение для автоматизации процессов выбранной предметной области с использованием данных, представленных Вами в схеме. Разработку вести на Ruby-on-Rails в общем гит-репозитории. Подготовить диаграмму классов всех сущностей предметной области.

Требования на оценку:

— Троечка —

1) Обязательно операции CRUD всех сущностей. (диаграмма последовательностей, хотя бы одна)

2) Обязательно отображение сущностей с внешними ключами для выбранного значения внешнего ключа. - (диаграмма последовательностей для отображения)

3) обязательна валидация полей сущности

— Не троечка —

4) 3 содерательных отчёта на отдельных вкладках с использованием всех таблиц

5) содержательная валидация с просмотром по нескольким сущностям

6) обработка исключительных ситуаций

7) бекап в файлы (JSON или YAML) и восстановление из файла

8) Работа с файлом бекапа, как с БД, в случае отключения БД

9) CTRL+Z

10) Выбрать один из 22 паттернов, кроме использованных в предыдущих пунктах и обоснованно его применить для реализации доп.фичи

11) Выбрать один из 22 паттернов, кроме использованных в предыдущих пунктах и обоснованно его применить для реализации доп.фичи

Правила разработки

Комментарии, длиннее одной строки в 100 символов - запрещены
