# Warehouse Scanner

Система для автоматизации складских операций: Android-приложение для сканирования штрихкодов и Spring Boot BFF-сервер, который связывает мобильный клиент со складским бэкендом.

> Приложение для работников склада: приёмка товара, инвентаризация и списание через сканирование штрихкодов камерой телефона.

## Архитектура

```
Android (Jetpack Compose)  ──HTTPS/JWT──▶  BFF (Spring Boot)  ──REST──▶  Backend (Rails)
       сканирование                         auth + агрегация              источник данных
```

Мобильный клиент не ходит в бэкенд напрямую. Между ними стоит **BFF (Backend for Frontend)** на Spring Boot: он отвечает за аутентификацию, держит свою модель складских сессий и проксирует запросы к основному бэкенду, отдавая клиенту ровно те данные, которые ему нужны.

## Стек

**Backend (BFF):**
- Java, Spring Boot, Spring MVC
- Spring Security + JWT (jjwt) — stateless-аутентификация
- JPA / Hibernate, PostgreSQL
- RestClient — синхронная интеграция с внешним бэкендом

**Mobile (Android):**
- Kotlin, Jetpack Compose
- CameraX + ML Kit — сканирование штрихкодов
- Retrofit + OkHttp, Koin (DI), DataStore (хранение токена)

## Возможности

- **Три режима работы со складом:** приёмка (RECEIVE), инвентаризация (INVENTORY), списание (WRITE_OFF) — логика количества зависит от режима.
- **Сканирование штрихкодов** камерой в реальном времени через ML Kit, с распознаванием товара по коду.
- **JWT-аутентификация:** логин по учётным данным сотрудника, токен автоматически подставляется во все запросы через интерсептор.
- **Интеграция с внешним бэкендом:** поиск товара по штрихкоду и обновление остатков проксируются через BFF.

## Запуск

### BFF (Spring Boot)

Сервер поднимется на `http://localhost:8080`.

### Android-клиент

Открой проект в Android Studio, укажи адрес BFF в конфигурации и запусти на устройстве/эмуляторе с камерой.

## Структура BFF

Ключевые компоненты серверной части:

- `JwtService`, `JwtAuthFilter`, `SecurityConfig` — аутентификация и stateless-сессии
- `RestClient`-интеграция — общение с внешним бэкендом
- JPA-сущности: `Employees`, `WarehouseSessions`, `SessionItems`, `InventoryCampaigns`

---

*Учебный проект. Серверная (BFF) и мобильная части написаны мной; складской бэкенд — отдельный сервис команды.*
